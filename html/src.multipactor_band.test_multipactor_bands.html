merge(union: str, name: str = '', filter_out_none: bool = True, info_test: str = '', several_bands_politics: str = 'merge') → InstrumentMultipactorBands

Merge the InstrumentMultipactorBands in self.

For that, we merge their multipactor boolean numpy array and recreate a InstrumentMultipactorBands with its own MultipactorBand.

Todo

Put a flag that will check consistency of position of MP bands. Like: assert_instrument_multipactor_bands_detected_at_same_position: bool.

Parameters:

  • union ({'strict', 'relaxed'}) – How the multipactor zones should be merged. It ‘strict’, all instruments must detect multipactor to consider that multipactor happened. If ‘relaxed’, only one instrument suffices.

  • name (str, optional) – Name that will be given to the returned InstrumentMultipactorBands. The default is an empty string, in which case a default meaningful name will be given.

  • filter_out_none (bool, optional) – To remove the None in self. The default is True.

  • info_test (str, optional) – To give more explicit output when there is a problem in the merging process. The default is an empty string.

  • several_bands_politics ({'keep_first', 'keep_last', 'keep_all', 'merge'}) –

    What to to when several multipactor bands are found in the same half-power cycle:

    • 'keep_first': we keep first MultipactorBand

    • 'keep_last': we keep last MultipactorBand

    • 'keep_all': we keep all MultipactorBand

    (currently not implemented) - 'merge': the final MultipactorBand spans from start of first MultipactorBand to end of last.

Returns:

instrument_multipactor_bands – Object holding merged multipactor bands.

Return type:

InstrumentMultipactorBands

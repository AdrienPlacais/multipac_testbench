src.multipactor_band.util module

Ensure that MultipactorBands are consistently used.

match_with_mp_band(obj: Instrument | Sequence[Instrument] | IMeasurementPoint | Sequence[IMeasurementPoint], multipactor_bands: MultipactorBands | Sequence[MultipactorBands], assert_positions_match: bool, find_matching_pairs: bool = False, **matching_pair_kw: bool) → zip
match_with_mp_band(obj: Sequence[Instrument] | Sequence[IMeasurementPoint], multipactor_bands: Sequence[MultipactorBands], assert_positions_match: bool, find_matching_pairs: bool = True, **matching_pair_kw: bool) → zip

Zip instruments with multipactor_bands.

Perform some checkings, to consistency between the instruments under study and the MultipactorBands object.

Parameters:

  • obj (Instrument | Sequence[Instrument] | IMeasurementPoint | Sequence[IMeasurementPoint]) – Objects for which you need to find multipactor.

  • multipactor_bands (MultipactorBands | Sequence[MultipactorBands]) – Objects holding info on when multipactor appears. If there is only one MultipactorBands, it will be applied on all the obj.

  • assert_positions_match (bool, optional) – To check if position where multipactor was checked must match the position of the instruments. The default is True.

  • find_matching_pairs (bool, optional) – Use this to ensure that for every item of obj, there is a matching MultipactorBands. Use this when there is a length mismatch, or when one of the items of obj does not have a MultipactorBands. The default is False.

Returns:

Object storing matching pairs of obj item with MultipactorBands.

Return type:

zip

positions_match(obj: Instrument | IMeasurementPoint, multipactor_bands: MultipactorBands, tol: float = 1e-06) → bool

Check that positions of obj and multipactor_bands are consistent.

Parameters:

  • obj (Instrument | IMeasurementPoint) – An object with a position attribute. It it is np.NaN, it means that the object under study is “global” and we return True.

  • multipactor_bands (MultipactorBands) – The multipactor bands to check. If its position is np.NaN, it means that the multipactor is detected at the scale of the whole testbench. In this case, we return True.

  • tol (float, optional) – Tolerance over the position matching. The default is 1e-6.

Returns:

If the positions of obj and multipactor_bands match or not.

Return type:

bool
